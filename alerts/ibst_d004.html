Runtime overload: Task critical to IO runtime (x1 Task) exceeded the deadline. 

The monitoring checks for the real execution time of critical processes in the IO critical task (x1 Task) versus the ideal start time with an allowed jitter (5% for non-iBooster Systems, 10% for iBooster Systems). The jitter limit of the restricted part is not held (x1 task). Possible causes of this include higher priority tasks taking longer, longer runtime in ISR, and X1 task being overloaded.
